What is WoT - VA?

(W)eb (o)f (T)hings - (V)ertical (Integration)

In the „Geospatial Web of Things“ study project the idea of the internet of things was applied for mobile water gauges. All together there were three groups working on the project, each with a different approach. Our group, the “Physical Smart Thing”-group (also called the Vertical Integration-Group), had a focus on implementing all smart capabilities on the thing itself.

This webside is the result of the combinded effort of all group members. In the hope that this piece of Software will be used one day to prevent surprising floods, we build page with much effort. Even though we did our best to programm without any erros, there might be still some undected ones. If you find any, feel free to fork our code at https://github.com/mrunde/WoT-Vertical-Approach and enhance it as you like!


Greetings: Moritz Migge, Marius Runde, Daniel Ummelmann & Axel Virnich



Running the Web Server

Before you start to install the seperate components you have to fork our repository.
Enter the following line of code into your command prompt to do so:

git clone https://github.com/mrunde/WoT-Vertical-Approach.git

Afterwards perform the following steps:

  • Install all node modules with npm install
  • Install bower, gulp and pm2 globally, if you haven't done so before with npm install -g bower gulp pm2
  • Install all bower modules with bower install
  • Perform the gulp tasks with gulp
  • Then you can start the web server with pm2 start server\server.js
In case the API documentation is not generated at the first time, please run the gulp task again.

How to setup a smart thing (Raspberry Pi 3)

This is an Installation-Guide for the Architecture of a Raspberry Pi 3.

Running the SensorController from Eclipse IDE

  • Instructions for Raspberry Pi
    • Download Java ME Embedded from the Oracle Website and install it on your Raspberry Pi.
    • Run Java ME Embedded on the Pi. Navigate to your installation directory and run sudo .\usertest.sh .
  • Instructions for Eclipse IDE
    • Install the Oracle Java ME SDK on your computer. You can find the SDKs on the Oracle Website.
    • In Eclipse, install the Oracle Java ME SDK Eclipse Plugin
    • Change to the Java ME Perspective by selecting "Window -> Perspective -> Open Perspective -> Java ME".
    • Import the SensorController as Java ME Project
    • Make sure the Raspberry Pi is connected to your computer via the Device Connections Manager. If your device is not yet connected, add the connection with the Device Connections Manager. The Device Connections Manager is installed with the Oracle Java ME SDK. You can add connections by selecting Add new device connection....
    • In Eclipse, change the execution environment to your Pi. Open the Application Descriptor and in the Overview Tab select "Add... -> Manage Devices... -> Manual Install...". In the Manual Device Installation Dialog select Browse and browse to your Java ME SDK installation directory. Close the Preferences Dialog and in the Device Selection of the Add Configuration Dialog select your Raspberry Pi.
    • If you start the SensorController now, it will be temporarily installed and executed on the Raspberry Pi.
  • Further assistance:
    • Refer to this tutorial if you have problems installing Java ME Embedded on the Raspberry Pi.
    • Refer to this tutorial if you want to learn more about developing with the Java ME SDK or if you are using NetBeans.
    • Refer to this tutorial if you have problems connecting the Raspberry Pi to your IDE or if you are using NetBeans.

Deploying the SensorController on the Raspberry Pi

  • Export the SensorController as MIDlet Package. If you are using Eclipse, you can use the Export Dialog.
  • Move the resulting .jar and .jad files to your Raspberry Pi, for example by using pscp .
  • Install the SensorController by running [path to your Java ME directory]/installSuite.sh [path to SensorController.jar] . If the installation is successful, a MIDlet ID is returned.
  • You can now run the SensorController by typing [path to your Java ME directory]/runSuite.sh [MIDlet ID] . If you forget the MIDlet ID you can check all installed MIDlets by running [path to your Java ME directory]/listMidlets.sh .
  • The SensorController will start up.

REST API

Here you can see an overview of the available REST API functions.
To see the complete Rest API Documentation click here.

Things

Status URL Method Description
api/things GET, POST all things
api/things/:thingId DELETE, GET one thing
api/things/:thingId/sensors GET all sensors of one thing
api/things/:thingId/measurements GET all measurements of one thing
api/things/measurements/latest GET latest measurements of all things
pi/things/:thingId/measurements/latest GET latest measurements of one thing
api/things/:thingId/nearest GET nearest other thing
api/things/feature/:featureId GET all things which measure this feature
api/things/temporal/:dateFrom/:dateTo GET all things within one time frame
api/things/spatial/bbox/:bbox GET all things within one bounding box
api/things/spatial/waterbodies/:waterbodyId GET all things next to one waterbody
api/things/temporal/:dateFrom/:dateTo/spatial/:bbox GET all things within one time frame and one bounding box

Sensors

Status URL Method Description
api/sensors GET, POST all sensors
api/sensors/:sensorId DELETE, GET, PUT one sensors
api/sensors/:sensorId/measurements GET all measurements of one sensor
api/sensors/temporal/:dateFrom/:dateTo GET all sensors within one time frame
api/sensors/spatial/:bbox GET all sensors within one bounding box
api/sensors/temporal/:dateFrom/:dateTo/spatial/:bbox GET all sensors within one time frame and one bounding box

Measurements

Status URL Method Description
api/measurements GET, POST all measurements
api/measurements/:measurementId DELETE, GET one measurement
api/measurements/temporal/:dateFrom/:dateTo GET all measurements within one time frame
api/measurements/spatial/:bbox GET all measurements within one bounding box
api/measurements/temporal/:dateFrom/:dateTo/spatial/:bbox GET all measurements within one time frame and one bounding box

Users

Status URL Method Description
api/users POST one user
api/things/:thingId DELETE, GET, PUT one user
api/users/:userId/things GET all things of one user

Features

Status URL Method Description
api/features GET, POST all features
api/features/:featureId GET one feature

Waterbodies

Status URL Method Description
api/waterbodies GET all waterbodies
api/waterbodies/:waterbodyId GET one waterbody

Why are we using Twitter
